Blocking such content can be a nightmare for website owners. This feature is only useful if the website is requesting that the user input their information through an insecure form. However, Chrome also blocks "insecure" HTTP content even if it does not ask the user for any input. This harms the user's experience, as they may not be able to load certain resources that contain useful information or provide the help they need.


Please consider modifying this technology so that it only blocks content that asks for the user's input. Until then, we will continue to refer our visitors to alternate browsers so they can see our content, as I am sure other webmasters are doing as well.
